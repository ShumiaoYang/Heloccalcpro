# AI工具站快速启动模板 - 项目规划纪要 (v1.0)
## 1. 项目目标 (Why)
- 要解决的核心问题： 赋能开发者（尤其是一人公司或小团队），使其在发现有价值的AI应用方向时，能够跳过认证、支付、部署等重复性基础工作，用最短的时间将想法上线验证。
- 目标用户： 一人公司、独立开发者或小型创业团队。
- 核心价值： 极大地提升AI工具从创意到上线的 速度 和 效率 ，降低市场验证的成本。
## 2. 核心需求 (What)
- 核心功能模块：
  - 基础框架： 基于Next.js构建，支持在Vercel平台一键部署。
  - 页面布局： 实现单页滚动的响应式布局，包含导航栏、Hero、工具、特性、博客、页脚等可配置区域。
  - SEO友好： 支持对站点的 Title , Description , Keywords 以及页面内的 H1 , H2 , H3 标签内容进行完全配置。
  - 核心工具区 (Tool Section)： 提供一个简化的、可扩展的交互模板。开发者可以基于此模板，通过少量定制开发，快速适配自己的AI工具（如文本生成、图文生成等）。
  - LLM调用封装： 后端提供统一的API接口，用于调用各大语言模型。支持通过配置安全地管理不同模型的 API Key 。
  - 用户认证系统： 支持用户通过Google等第三方账号进行注册和登录。
  - 支付与订阅系统： 集成Stripe和Creem支付网关，支持创建不同的订阅套餐（如免费版、专业版），并根据用户等级控制其功能权限和使用额度。
  - 多语言支持 (i18n)： 框架支持国际化，可通过URL路径（如 site.com/zh ）切换不同语言。
- 关键体验要求：
  - 视觉风格： 追求“极致简约的科技感”，同时采用“柔和讨喜”的色彩方案，避免纯黑白的单调，提升亲和力。
  - 模板易用性： 采用“小而美”的设计哲学，保持模板的轻量和高扩展性，让开发者能快速上手进行二次开发。
## 3. 可行性与技术选型 (If & How)
- 主要技术挑战： 设计一个足够抽象和灵活的核心工具区（Tool Section）组件，使其能够通过简单的配置或少量代码，就能快速适应不同AI工具的前端交互需求。
- 初步技术方案：
  - 核心框架： Next.js
  - UI构建： Tailwind CSS + shadcn/ui
  - 国际化 (i18n)： next-intl
  - 用户认证： NextAuth.js
  - 支付集成： Stripe SDK, Creem API
  - LLM调用： 在Next.js后端创建API Routes进行统一封装和管理。
- 决策理由： 该技术栈是当前构建高性能Web应用的成熟方案，与Vercel生态无缝集成，社区活跃，资源丰富。选择“小而美”的策略是为了优先保证开发速度和模板的易用性。
## 4. MVP 版本规划 (MVP)
- MVP核心目标： 以最快的速度、最低的成本上线一个可用的AI工具站点，验证 核心AI功能本身的市场吸引力 ，并跑通从开发到Vercel部署的全流程。
- MVP包含功能：
  - 1. 核心页面框架： 实现包含导航栏、Hero区、简化的工具区和页脚的基础布局。
  - 2. 单一AI工具实例： 实现一个具体的、简单的AI工具（如“AI口号生成器”），打通前端输入 -> 后端调用LLM -> 前端展示结果的完整链路。
  - 3. 核心SEO配置： 实现 Title , Description , Keywords , H1 , H2 , H3 的可配置化。
  - 4. 基础配置能力： 支持在环境变量中配置LLM的 API Key 。
  - 5. 一键部署： 确保项目代码可以被顺利部署到Vercel。
- 后续迭代方向（非MVP功能）：
  - 用户认证系统： 增加基于 NextAuth.js 的Google账号登录功能。
  - 分级订阅与支付： 在用户系统的基础上，集成Stripe和Creem，并开发订阅管理和权限控制逻辑。
  - 多语言支持： 集成 next-intl ，提供中英文切换功能。
  - 丰富页面模块： 逐步实现并完善 Features , Blog , Example 等其他页面板块。
  - 高级工具区模板： 开发更复杂的工具区模板，以适应如图文生成等多模态交互场景。
